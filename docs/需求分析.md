# 需求分析

## 1. 项目概述

### 1.1 项目背景

高校教学中同一学科存在多本教材并行使用现象，教材间存在大量内容重复，对学生、教师和教学管理者造成困扰：
- 学生：重复阅读浪费时间
- 教师：难以评估教材交叉与缺失
- 管理者：缺乏课程整合依据

### 1.2 项目目标

开发学科知识整合智能体，将7本教材整合压缩到不超过原始体量30%，且教学效果不打折。

### 1.3 核心功能

- 多格式教材加载与解析
- 单本教材知识图谱构建与可视化
- 知识图谱交互
- 跨教材知识图谱整合
- RAG精准问答
- Agent架构设计
- 整合建议与多轮对话
- Web交互界面

## 2. 子问题分解

### 2.1 教材解析问题

| 子问题 | 解决方案 |
|--------|----------|
| 多格式支持 | PDF用PyMuPDF、DOCX用python-docx、TXT/MD直接读取 |
| 章节识别 | 正则匹配"第X章"标题 + 字体大小/加粗判断 |
| 页眉页脚过滤 | 识别重复出现的页眉页脚模式并跳过 |
| 大文件处理 | 逐页解析，流式处理，避免OOM |

### 2.2 知识提取问题

| 子问题 | 解决方案 |
|--------|----------|
| 知识点识别 | LLM提取核心概念、定理、方法、现象 |
| 关系抽取 | LLM识别前置依赖、并列、包含、应用关系 |
| 结构化输出 | Prompt约束JSON格式输出 |

### 2.3 跨教材整合问题

| 子问题 | 解决方案 |
|--------|----------|
| 语义对齐 | Embedding相似度 + LLM双重判断 |
| 去重决策 | merge/keep/remove三类决策 |
| 压缩控制 | 统计字数，确保≤30% |

## 3. 知识点粒度定义

### 3.1 知识点判定标准

一个知识点应满足：
1. **独立性**：可单独理解和讲解的原子概念
2. **完整性**：有明确定义或描述
3. **教学价值**：对学生学习有实际意义

### 3.2 知识点类型

| 类型 | 定义 | 示例 |
|------|------|------|
| 核心概念 | 学科基础概念 | 动作电位、静息电位 |
| 定理/原理 | 具有普遍性的规律 | 能量守恒定律 |
| 方法/技术 | 具体操作方法 | PCR技术、Western Blot |
| 现象/过程 | 动态变化过程 | 有丝分裂、炎症反应 |

### 3.3 粒度边界

- **太粗**："细胞生物学"（应拆分为多个子概念）
- **太细**："细胞的定义中第一个词"（无独立教学价值）
- **合适**："细胞膜的结构与功能"（可独立讲解）

## 4. 重复判定标准

### 4.1 语义相似度阈值

| 阈值 | 判定结果 | 说明 |
|------|----------|------|
| ≥0.85 | 高度重复 | 几乎同一概念，合并 |
| 0.70-0.85 | 可能重复 | LLM二次判断 |
| 0.50-0.70 | 相关但不同 | 保留各自版本 |
| <0.50 | 独立概念 | 不处理 |

### 4.2 阈值设定理由

选择0.85作为高置信阈值：
- BGE-small-zh模型在中文语义匹配任务上表现稳定
- 过高阈值（0.90+）会漏掉表述差异大的重复概念
- 过低阈值（0.80以下）会误合并相关但不同的概念

### 4.3 LLM二次判断Prompt

当相似度在0.70-0.85区间时，调用LLM判断：

```
判断以下两个知识点是否为同一概念：
知识点A：{name_a}，定义：{definition_a}
知识点B：{name_b}，定义：{definition_b}

判断标准：
1. 核心含义是否相同
2. 是否只是表述方式不同
3. 是否存在实质性的内容差异

输出JSON：{"is_same": true/false, "reason": "判断理由"}
```

## 5. 教学连贯性保障

### 5.1 知识链路检测

整合后需检测：
- 前置关系链是否断裂（学习B需要先掌握A，但A被删除）
- 知识覆盖是否完整（课程大纲要求的知识点是否保留）

### 5.2 保障策略

1. **保留关键节点**：被多个知识点依赖的概念不删除
2. **合并时保留完整版本**：选择描述最系统的版本
3. **人工确认机制**：关键整合决策需教师确认

---

## 6. 压缩比计算方式

### 6.1 计算公式

```
压缩比 = 整合后总字数 / 原始总字数 × 100%
```

### 6.2 字数统计范围

- 仅统计正文内容（不含页眉页脚、图表标注）
- 整合后统计合并后的知识点定义和关系描述

### 6.3 目标控制

- 必须 ≤30%
- 建议 20-25%（留有安全余量）

---

## 7. RAG分块策略

### 7.1 分块参数

| 参数 | 值 | 理由 |
|------|-----|------|
| 块大小 | 600字 | 平衡上下文完整性和检索精度 |
| 重叠区 | 80字 | 防止知识点被截断 |
| 元数据保留 | 教材名、章节、页码 | 支持来源引用 |

### 7.2 分块策略选择依据

| 策略 | 优点 | 缺点 | 选择 |
|------|------|------|------|
| 300字小块 | 检索精度高 | 上下文不完整 | 不采用 |
| 600字中块 | 平衡精度与完整性 | - | **采用** |
| 1000字大块 | 上下文完整 | 检索噪音多 | 不采用 |

### 7.3 中文Embedding选择

选择 **BGE-small-zh**：
- 中文语义理解能力强
- 模型体积小（约100MB），部署快
- 无需API调用费用
